# SSE + REST Protocol

klaus uses **Server-Sent Events** (SSE) for real-time server-to-client streaming and **REST** endpoints for client-to-server actions.

## SSE Stream

```
GET /api/events/stream?session_id=<session-id>
```

Returns a `text/event-stream` response. On connect, replays the last 50 events from history, then streams live events.

### Event Format

Each event is sent as a standard SSE `data:` line containing JSON:

```
data: {"type": "chat.token", "data": {"token": "Hello", "chat_id": "123"}, "ts": 1717000000.0}

```

A keepalive comment (`: keepalive`) is sent every 30 seconds to prevent timeouts.

### Reconnection

`EventSource` provides built-in auto-reconnection. The `useEventStream` hook tracks connection state automatically.

## REST Endpoints (Client → Server)

### Send a Chat Message

```
POST /api/events/chat/send
Content-Type: application/json

{
  "messages": [{ "role": "user", "content": "How do I sort a list in Python?" }],
  "id": "session-uuid",
  "images": [],
  "model": null,
  "backend": null,
  "temperature": 0.7,
  "retry": false
}
```

**Response:** `{ "status": "ok", "chat_id": "session-uuid" }`

The response is returned immediately. The actual generation streams via SSE.

### Plan Approval / Rejection / Edit

```
POST /api/events/chat/{chat_id}/plan-action
Content-Type: application/json

{
  "action": "approve"
}
```

Actions: `approve`, `reject`, `edit`

For `reject`:

```json
{
  "action": "reject",
  "reason": "The plan misses the testing step"
}
```

For `edit`:

```json
{
  "action": "edit",
  "edits": [
    { "index": 0, "description": "Updated step description" },
    { "index": 2, "remove": true }
  ],
  "reason": "Simplified the workflow"
}
```

## Server → Client Events (via SSE)

### `chat.token`

Streaming token from the LLM:

```json
{ "type": "chat.token", "data": { "token": "Here", "chat_id": "123" }, "ts": 1717000000.0 }
```

### `chat.done`

Response complete:

```json
{ "type": "chat.done", "data": { "chat_id": "123" }, "ts": 1717000000.0 }
```

### `chat.error`

An error occurred:

```json
{ "type": "chat.error", "data": { "error": "Failed to connect to Ollama", "chat_id": "123" }, "ts": 1717000000.0 }
```

### `chat.status`

Progress feedback during processing:

```json
{ "type": "chat.status", "data": { "step": "routing", "detail": "Selecting best model...", "chat_id": "123" }, "ts": 1717000000.0 }
```

### `model.routed`

Routing decision for a request:

```json
{ "type": "model.routed", "data": { "backend": "ollama", "model": "granite-code:8b", "reason": "coding task", "chat_id": "123" }, "ts": 1717000000.0 }
```

### `mcp.tool_called`

A tool is being invoked:

```json
{ "type": "mcp.tool_called", "data": { "name": "remember", "args": { "path": "python/tips", "content": "..." }, "chat_id": "123" }, "ts": 1717000000.0 }
```

### `tool.result`

A tool has returned:

```json
{ "type": "tool.result", "data": { "name": "remember", "content": "Stored at /knowledge/python/tips", "chat_id": "123" }, "ts": 1717000000.0 }
```

### Orchestration Events

| Event | Description |
|-------|-------------|
| `plan.created` | Multi-step plan generated by orchestrator |
| `plan.awaiting_approval` | Plan is waiting for user approval |
| `plan.approved` | User approved the plan |
| `plan.rejected` | User rejected the plan |
| `plan.revised` | Plan was edited and updated |
| `plan.step_start` | Execution of a plan step begins |
| `plan.step_done` | A plan step completed |
| `plan.consolidated` | All steps done, consolidation starting |

## Message Flow

```
Client                                Server
  │                                     │
  │── POST /chat/send ─────────────────▶│  ← returns immediately
  │                                     │
  │◀─── SSE: chat.status (classifying) ─│
  │◀─── SSE: chat.status (routing) ─────│
  │◀─── SSE: model.routed ──────────────│
  │◀─── SSE: chat.status (generating) ──│
  │◀─── SSE: mcp.tool_called ───────────│  tool invocation
  │◀─── SSE: tool.result ───────────────│  tool result
  │◀─── SSE: chat.token ────────────────│  streaming tokens...
  │◀─── SSE: chat.token ────────────────│
  │◀─── SSE: chat.done ─────────────────│  response complete
  │                                     │
```
