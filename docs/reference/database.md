# Database Schema

klaus uses **PostgreSQL 17** with the **pgvector** extension for vector similarity search.

## Setup

The database is managed via Podman:

```bash
bash scripts/start-postgres.sh
```

This builds the `klaus-postgres` image from `Containerfile.postgres` and creates the `klaus-pg` container.

## Connection

| Setting | Default |
|---------|---------|
| Host | `localhost` |
| Port | `5432` |
| Database | `klaus` |
| User | `klaus` |
| Password | `klaus` |
| URL | `postgresql://klaus:klaus@localhost:5432/klaus` |

The Python backend uses `asyncpg` for async PostgreSQL access with a connection pool.

## Tables

### `memory_tree`

Stores the complete memory tree as a single JSONB document.

```sql
CREATE TABLE IF NOT EXISTS memory_tree (
    id       INTEGER PRIMARY KEY CHECK (id = 1),
    data     JSONB NOT NULL,
    saved_at DOUBLE PRECISION NOT NULL
);
```

The `CHECK (id = 1)` constraint ensures there's exactly one row — the entire tree state.

### `conversations`

Stores chat messages across sessions.

```sql
CREATE TABLE IF NOT EXISTS conversations (
    id         SERIAL PRIMARY KEY,
    session_id TEXT NOT NULL,
    role       TEXT NOT NULL,           -- 'user', 'assistant', 'system', 'tool'
    content    TEXT NOT NULL,
    model      TEXT,                     -- Model used for assistant responses
    backend    TEXT,                     -- Backend used
    created_at DOUBLE PRECISION NOT NULL -- Unix timestamp
);

CREATE INDEX IF NOT EXISTS idx_conversations_session
    ON conversations(session_id, created_at);
```

### `routing_rules`

Persists user-defined task routing rules.

```sql
CREATE TABLE IF NOT EXISTS routing_rules (
    task       TEXT PRIMARY KEY,
    rule       JSONB NOT NULL,           -- {"preferred_backend": "...", "preferred_model": "..."}
    updated_at DOUBLE PRECISION NOT NULL -- Unix timestamp
);
```

### `embeddings`

Stores vector embeddings for semantic search via pgvector.

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE IF NOT EXISTS embeddings (
    id         SERIAL PRIMARY KEY,
    path       TEXT NOT NULL,            -- Memory tree path (e.g., "/knowledge/python/tips")
    content    TEXT NOT NULL,            -- Original text content
    embedding  vector(384) NOT NULL,     -- all-MiniLM-L6-v2 embedding
    created_at DOUBLE PRECISION NOT NULL -- Unix timestamp
);

CREATE INDEX IF NOT EXISTS idx_embeddings_path
    ON embeddings(path);
```

The 384-dimensional vectors are generated by `sentence-transformers/all-MiniLM-L6-v2`.

## pgvector Operations

### Semantic search (cosine distance)

```sql
SELECT path, content,
       1 - (embedding <=> $1::vector) AS similarity
  FROM embeddings
 ORDER BY embedding <=> $1::vector
 LIMIT 5;
```

The `<=>` operator computes cosine distance. `1 - distance` gives cosine similarity.

### Upsert embeddings

When a memory node is updated, existing embeddings for that path are deleted and re-inserted:

```python
await conn.execute("DELETE FROM embeddings WHERE path = $1", path)
await conn.execute(
    "INSERT INTO embeddings (path, content, embedding, created_at) VALUES ($1, $2, $3, $4)",
    path, content, json.dumps(vector), time.time(),
)
```

## Backup & Restore

```bash
# Backup
podman exec klaus-pg pg_dump -U klaus klaus > backup.sql

# Restore
podman exec -i klaus-pg psql -U klaus klaus < backup.sql
```

## Podman Compose

The `docker-compose.yml` (used via `podman-compose`) includes PostgreSQL with a health check:

```yaml
services:
  postgres:
    build:
      context: .
      dockerfile: Containerfile.postgres
    ports:
      - "5432:5432"
    environment:
      POSTGRES_DB: klaus
      POSTGRES_USER: klaus
      POSTGRES_PASSWORD: klaus
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U klaus"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  pgdata:
```
